# 오늘응급 화면/UX 설계

## 1. UX 원칙

1. 응급 상황에서는 탐색보다 행동이 빨라야 한다.
2. 전화 버튼은 항상 눈에 띄어야 한다.
3. 불확실한 정보는 숨기지 않고 `전화 확인 필요`로 표시한다.
4. 색상만으로 상태를 전달하지 않는다.
5. 고령자와 보호자도 쓸 수 있도록 큰 글씨와 큰 버튼을 쓴다.
6. 생명이 위급한 상황에서는 앱보다 119가 우선임을 안내한다.

## 2. IA

```text
홈
├─ 가까운 응급실
│  ├─ 응급실 목록
│  ├─ 지도 보기
│  └─ 응급실 상세
├─ 지금 문 연 약국
│  ├─ 약국 목록
│  ├─ 지도 보기
│  └─ 약국 상세
├─ 주소/지역 검색
└─ 긴급 전화
   ├─ 119
   └─ 129
```

## 3. 홈 화면

### 목적

앱 실행 후 1~2번의 터치로 응급실/약국 검색 또는 119 전화가 가능해야 한다.

### 구성

```text
[오늘응급]
내 주변 응급실과 문 연 약국을 빠르게 찾으세요.

[가까운 응급실 찾기]
[지금 문 연 약국 찾기]

[119 전화하기]
[지역/주소로 검색]

작은 안내:
응급 상황이면 즉시 119에 신고하세요.
```

### CTA 우선순위

1. 119 전화하기
2. 가까운 응급실 찾기
3. 지금 문 연 약국 찾기
4. 지역/주소로 검색

## 4. 응급실 목록 화면

### 카드 구성

```text
[상태 배지] 전화 확인 필요
서울OO병원 응급실
1.2km · 권역응급의료센터
서울특별시 ...
응급실 전화: 02-0000-0001

[전화하기] [길찾기] [상세]
```

### 정렬

- 기본: 거리순
- 추후: 상태 우선 + 거리순

### 필터

MVP:

- 반경: 1km / 3km / 5km / 10km
- 지도 보기

2차:

- 소아 가능
- 중증질환 수용 가능
- 응급의료기관 분류

## 5. 약국 목록 화면

### 카드 구성

```text
[상태 배지] 현재 운영 중
OO온누리약국
850m
운영시간: 09:00~22:00
서울특별시 ...
전화: 02-0000-0000

[전화하기] [길찾기] [상세]
```

### 상태 표시

- 현재 운영 중
- 운영 종료
- 전화 확인 필요
- 정보 없음

## 6. 지도 화면

### 구성

```text
[지도]
  - 현재 위치
  - 응급실 마커
  - 약국 마커

하단 카드:
기관명 / 거리 / 전화 / 길찾기
```

### 마커

- 응급실: 빨간색 십자
- 약국: 초록색 알약
- 내 위치: 파란색 점

## 7. 상세 화면

### 응급실 상세

- 병원명
- 상태 배지
- 주소
- 대표전화
- 응급실 전화
- 병상/수용 가능 정보
- 지도
- 데이터 출처
- 갱신 시각
- 고지 문구
- 하단 고정 전화 버튼

### 약국 상세

- 약국명
- 운영 상태
- 운영시간
- 주소
- 전화번호
- 지도
- 데이터 출처
- 갱신 시각
- 고지 문구
- 하단 고정 전화 버튼

## 8. 위치 권한 흐름

### 허용 시

1. 현재 좌표 획득
2. 검색 API 호출
3. 거리순 목록 표시

### 거부 시

```text
위치 권한이 꺼져 있습니다.
주소나 지역명으로 검색할 수 있습니다.

[주소/지역 입력]
[검색]
```

## 9. 오류 화면

### API 실패

```text
현재 정보를 불러오지 못했습니다.
공공데이터 응답이 지연되고 있을 수 있습니다.
방문이 급하면 119 또는 기관에 직접 전화하세요.

[다시 시도]
[119 전화하기]
```

### 검색 결과 없음

```text
주변에서 조건에 맞는 결과를 찾지 못했습니다.
검색 반경을 넓히거나 전화 상담을 이용하세요.

[반경 10km로 넓히기]
[119 전화하기]
```
