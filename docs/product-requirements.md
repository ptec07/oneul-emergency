# 오늘응급 제품 요구사항 문서

## 1. 목표

오늘응급은 야간·주말·공휴일 및 응급 상황에서 사용자가 현재 위치 기준으로 가까운 응급실과 운영 중인 약국을 빠르게 찾도록 돕는 PWA다.

## 2. 핵심 사용자 시나리오

### 시나리오 A: 야간에 아이가 아픈 보호자

1. 사용자가 오늘응급 PWA를 실행한다.
2. 현재 위치 권한을 허용한다.
3. `가까운 응급실`을 누른다.
4. 거리순 응급실 목록을 확인한다.
5. 응급실 전화 버튼으로 방문 가능 여부를 확인한다.
6. 길찾기로 이동한다.

### 시나리오 B: 공휴일에 문 연 약국이 필요한 사용자

1. 사용자가 오늘응급 PWA를 실행한다.
2. `지금 문 연 약국`을 누른다.
3. 운영 중 또는 전화 확인이 필요한 약국 목록을 확인한다.
4. 약국 전화 버튼으로 운영 여부를 확인한다.
5. 길찾기로 이동한다.

### 시나리오 C: 위치 권한을 거부한 사용자

1. 사용자가 위치 권한을 거부한다.
2. 주소/지역명 검색 화면을 제공한다.
3. 입력한 위치 기준으로 응급실/약국을 검색한다.

## 3. MVP 범위

### 포함

- PWA 설치 지원
- 현재 위치 기반 검색
- 주소/지역명 검색 대체 흐름
- 응급실 목록 및 상세
- 약국 목록 및 상세
- 전화하기
- 지도 앱 길찾기 연동
- 데이터 출처 및 갱신 시각 표시
- 전화 확인 필요 안내

### 제외

- 회원가입
- 의료 상담/AI 문진
- 예약/접수
- 리뷰/평점
- 결제
- 진료기록 저장
- 백그라운드 위치 추적

## 4. 기능 요구사항

### FR-001 홈 화면

- 사용자는 홈에서 `가까운 응급실`, `지금 문 연 약국`, `119 전화하기`를 볼 수 있어야 한다.
- 위치 권한 요청은 사용자의 명시적 액션 이후 실행한다.

### FR-002 응급실 검색

- 현재 위치 또는 입력 위치 기준 반경 내 응급의료기관을 조회한다.
- 결과는 거리순으로 정렬한다.
- 각 결과는 병원명, 거리, 주소, 응급실 전화, 상태, 길찾기 버튼을 포함한다.

### FR-003 약국 검색

- 현재 위치 또는 입력 위치 기준 약국을 조회한다.
- 현재 시간 기준 운영 여부를 계산한다.
- 운영시간 정보가 불명확하면 `전화 확인 필요`로 표시한다.

### FR-004 상세 화면

- 기관명, 주소, 전화번호, 위치, 운영/상태 정보, 데이터 출처, 갱신 시각을 표시한다.
- 상세 화면 상단 또는 하단에 큰 전화 버튼을 제공한다.

### FR-005 PWA

- 사용자는 모바일 브라우저에서 홈 화면에 앱을 추가할 수 있어야 한다.
- 앱 아이콘, 이름, 테마 색상, 시작 URL을 manifest에 정의한다.
- 기본 오프라인 화면을 제공한다.

## 5. 비기능 요구사항

### 속도

- 홈 화면 초기 렌더링: 2초 이내 목표
- 검색 결과 첫 표시: 3초 이내 목표
- API 실패 시 5초 이내 오류/재시도 안내

### 접근성

- 큰 버튼과 명확한 색상 대비
- 전화하기 버튼은 카드마다 노출
- 상태 색상만으로 의미 전달 금지, 텍스트 병행

### 신뢰성

- 공공데이터 API 장애 시 사용자에게 명확히 안내
- 최근 성공 응답 캐시를 단기 fallback으로 활용 가능
- 모든 의료 관련 상태에는 전화 확인 안내 표시

## 6. 성공 지표

- 전화 버튼 클릭률
- 길찾기 버튼 클릭률
- 검색 실패율
- API 오류율
- 재방문율
